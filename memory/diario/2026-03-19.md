## Diagnóstico — Bug 2: Triplicação no Slack

**Workflow:** `Monitoramento Suporte Evolution + IA + Slack (Template)`
**ID:** `lahexgzvNT2f3WRU`
**Status:** Ativo ✅

**Causa raiz (Nó "Filter + Normalize Event"):**
A lista de eventos permitidos inclui tipos que chegam múltiplos para cada mensagem:
```javascript
const allowed = [
  'MESSAGES_SET',
  'MESSAGES_UPSERT',    // ✅ mensagem nova — correto
  'MESSAGES_UPDATE',    // ❌ status entregue/lido — dispara de novo
  'CHATS_UPDATE',       // ❌ chat atualizado — dispara de novo
  'SEND_MESSAGE'        // ❌ confirmação de envio — dispara de novo
];
```

Uma única mensagem do cliente gera 3-4 eventos Evolution → workflow dispara 3-4x → Slack recebe 3-4 notificações idênticas.

**Agravante descoberto:**
O Monitoramento tem um nó `→ Agente Suporte IA` que faz POST para `/webhook/webhook-suporte-ia` (o agente). Como o Monitoramento dispara 3x, o agente também seria chamado 3x → quando o shadow mode for desativado, o cliente receberia a mesma resposta triplicada.

**Fix proposto:**
- Remover `MESSAGES_UPDATE`, `CHATS_UPDATE`, `SEND_MESSAGE` da lista `allowed`
- Manter apenas: `['MESSAGES_UPSERT', 'MESSAGES_SET']`
