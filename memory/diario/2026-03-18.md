## LIÇÕES APRENDIDAS

### F4 usa googleSheetsOAuth2Api para Drive
- F4 usa credencial `MXwK2XnNndg0AdQQ` (Google Sheets account) para chamar Drive API
- Não usa `googleDriveOAuth2Api` (`oRp4hBuwhcDDNu2p`) diretamente no HTTP Request
- O OAuth do Google tem escopos cruzados (Sheets + Drive no mesmo token)

### N8N API — workflow create é rígida
- `POST /api/v1/workflows` rejeita se faltar `settings` obrigatório
- `PUT /api/v1/workflows/{id}` rejeita `active` (read-only) e campos extras
- Melhor abordagem para modificar workflows: editar no painel visual, não via API
- Executar workflow via API: `POST /api/v1/workflows/{id}/run` funciona, mas requer workflow já configurado corretamente

### Google Drive upload direto (fora de N8N)
- Não é possível sem o token OAuth — ele fica interno ao N8N
- Para upload: precisa de workflow N8N ou acesso Docker ao container N8N

### ADVbox API — endpoints de documentos
- `POST /api/v1/customers/{id}/documents/` → 405 (não suportado)
- `POST /api/v1/lawsuits/{id}/files/` → 302 (redirect, não segue)
- Documentos ADVbox = somente via interface web (sem API de upload)

### Bug tipo processo F15 — default errado
- F15 usa `577852` (BAFÔMETRO) como default para TODOS os processos
