## [06:02 UTC] MF01 Pipeline — CONCLUÍDO ✅

### Problema raiz do dia
O nó `MF01 | OpenAI Chat Analisar Contrato` falhava com "Files not found" porque o `file_id` era enviado como `=file-xxx` (com `=` na frente). Causa: usar `={{ $json.id }}` dentro de uma string no parâmetro `jsonBody` do HTTP Request faz o n8n incluir o `=` literalmente.

### Fix definitivo
Padrão correto (igual ao MF05 que já funcionava):
- `specifyBody: "json"` + `contentType: "json"` + `jsonBody: "={{ JSON.stringify({...}) }}"`
- O `={{ }}` envolve o JS inteiro; dentro é JavaScript puro, sem conflito com o parser n8n

### Outros bugs corrigidos
1. **Conexão quebrada**: `PDF para Base64` → `OpenAI Analisar Contrato` (nome antigo após rename) — gerava "Cannot read properties of undefined (reading 'name')" no Webhook
2. **Binary strip**: Nó `PDF para Base64` convertia binário em JSON, quebrando o upload. Fix: conectar `Download PDF` → `OpenAI Upload PDF` direto
3. **Code node sandbox**: `require('https')` é bloqueado no n8n Code node. Solução: HTTP Request nodes para todas as chamadas externas
4. **Body como chave de formulário**: `specifyBody: "string"` + `body: "={{ $json.requestBody }}"` envia o JSON como KEY de form, não como raw body

### Arquitetura final do MF01 (cadeia PDF)
```
ZapSign GET → Download PDF → OpenAI Upload PDF → Build OpenAI Body (Code) → OpenAI Chat (HTTP Request) → Extrair Dados do Contrato
```
